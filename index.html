Retaining Results

Version control tracks source code, not results

Need somewhere to store results, e.g. a database

Also, prototyping is more free-form

Many tweaks aren't worth committing

Free-form solution: Electronic lab notebook

Enter FGLab

Node.js + MongoDB: Open source + cross-platform

Wraps around existing experiment code to retain results

Setup

FGLab on one node

MongoDB on one node (or several)

FGMachine(s) on compute node(s)

For distribution, FGLab and FGMachine(s)
must be HTTP-accessible

Single node setup is easy (use localhost)

Requirements

  • Permissions to run code on compute node(s)
  • Code accepts command-line inputs, outputs files
  • Node.js on compute node(s)

FGLab deployment: Local, Docker and Heroku

FGMachine deployment: Local and Docker

Workflow

Choose program options via FGLab GUI / HTTP API

Submit to available FGMachine

FGMachine spawns program, sends back results

JSON output parsed, other files stored
in MongoDB GridFS
